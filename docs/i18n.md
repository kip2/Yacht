# 国際化 (i18n)

`public/app.js` 内の **辞書ベース** の軽い仕組み。フレームワーク不要。
日本語 (デフォルト) と英語の 2 言語を切り替え可能。

## 仕組み

- `I18N = { ja: {...}, en: {...} }` をモジュール先頭に置く。
  値は文字列または `(args) => string` の関数。
- 取得は `t(key, ...args)` と `tCat(categoryKey)` / `tCatTip(categoryKey)`。
- 静的な要素は HTML に `data-i18n="key"` を付けておき、
  `applyStaticI18n()` が `querySelectorAll("[data-i18n]")` で textContent を一括置換。
- 動的に生成する要素 (ステータス・スコアボード・カテゴリボタン) は、
  各 `render*` 関数内で `t()` / `tCat()` を呼びながら組み立てる。
- 言語切替ボタン (`#btn-lang`) は `state.lang` をトグルして
  `applyStaticI18n()` + 既存の `render()` を再実行するだけ。
- **デフォルトは日本語** (`localStorage.getItem("yacht.lang") || "ja"`)。

## 新しい文言を追加するとき

`I18N.ja` と `I18N.en` の **両方に同じキー** を足す。片側だけだと
`t()` が `undefined` を返して画面に "undefined" と出るので、
不一致は実行画面で気づきやすい。

```js
const I18N = {
  ja: {
    ...
    myNewKey: "新しい文言",
    myNewFn: (n) => `${n} 個あります`,
  },
  en: {
    ...
    myNewKey: "New text",
    myNewFn: (n) => `${n} item(s)`,
  },
};
```

静的要素なら HTML に `data-i18n="myNewKey"` を付けるだけで自動的に流し込まれる。
動的に生成する場所では `el.textContent = t("myNewKey")` のように呼ぶ。

## 役名の追加・変更時の注意

役 (Category) を追加する場合は **3 箇所** にエントリを書く必要がある:

1. `I18N[lang].categories[キー]` — 表示名
2. `I18N[lang].categoryTips[キー]` — ホバー時の説明 (短文)
3. `I18N[lang].help.roles` — 「遊び方」モーダル一覧 (例つき長文)

D 側の `Category` enum (`source/game/category.d`) と
JS 側 `CATEGORY_KEYS` 配列の **インデックスを一致** させること。

## 言語追加 (例: フランス語)

`I18N` に `fr: { ... }` を追加し、`toggleLang()` を 2 言語トグルから
3 言語ローテーションに変える:

```js
const LANGS = ["ja", "en", "fr"];
function toggleLang() {
  const i = LANGS.indexOf(state.lang);
  state.lang = LANGS[(i + 1) % LANGS.length];
  ...
}
```

`langButton` の値は **遷移先の言語名** にする (現在 `ja → "EN"`, `en → "日本語"`)。
3 言語以上ならローテーション順を考慮して書き換える。
